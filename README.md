# 🌯 ЛавашОК — сайт доставки шаурмы + приём заказов

Готовый одностраничный лендинг с бэкендом на Node.js/Express: посетитель
собирает корзину, оформляет заказ (с оплатой при получении или онлайн через
ЮKassa), заявка сохраняется в базу и появляется в админке.

## Быстрый запуск (2 команды)

Нужен установленный [Node.js](https://nodejs.org) версии 18 или новее.

```bash
npm install
npm start
```

После запуска:

| Что          | Адрес                          |
|--------------|--------------------------------|
| Сайт         | http://localhost:3000          |
| Админка      | http://localhost:3000/admin    |

**Вход в админку** через страницу `/admin/login`, по умолчанию `admin` / `lavashok`
(смените перед публикацией — см. «Безопасность»).

> Демо за минуту: открой сайт → пролистай до «Оформить заказ» → отправь заявку →
> открой `/admin` и увидишь её в списке. Меняй статусы (Новый → Перезвонили →
> Готово) прямо в таблице.

## Возможности

**Фронтенд**
- адаптивная вёрстка (десктоп / планшет / телефон) с бургер-меню;
- меню с фильтром по категориям и **фото блюд** (загружаются из админки);
- корзина: добавление блюд, изменение количества, автосумма, **допродажи** («добавьте к заказу») и **прогресс-бар до бесплатной доставки**;
- оформление заказа с выбором оплаты (при получении / онлайн-картой);
- **трекинг заказа для клиента**: ссылка «Отследить заказ» со статусами принят → готовим → в пути → доставлен;
- форма с маской телефона, валидацией и состоянием «Отправляем…»;
- favicon и Open Graph — красивое превью ссылки в мессенджерах;
- автоподстановка UTM-меток из ссылки (для рекламы);
- заготовки целей Яндекс.Метрики (`order_click`, `phone_click`, `form_submit`).

**Бэкенд**
- `POST /api/order` — приём заказа с **валидацией (zod)**, **сумма считается на сервере по `lib/menu.js`** (клиенту не доверяем);
- защита от спама: **honeypot-поле** + **rate-limit** (не более 6 заявок с IP за 10 минут);
- хранилище заказов — **SQLite** (`better-sqlite3`) с автоматическим фолбэком на JSON-файл;
- безопасность: **helmet** (security-заголовки + CSP), вход в админку по **сессии** с **bcrypt-хешем пароля** и лимитом попыток входа;
- онлайн-оплата ЮKassa: создание платежа, возврат, статус, **вебхук с проверкой IP**;
- `GET /admin` (сессия): состав заказа, сумма, способ и статус оплаты; `/admin/login`, `/admin/logout`;
- `GET /api/orders`, `POST /api/orders/:id/status`, `POST /api/orders/:id/note` — список, статус, заметка;
- `GET /api/menu` (публичный) и `GET/PUT /api/admin/menu` — витрина и редактирование меню;
- `GET /health` — живость + используемый драйвер хранилища.

**Админка** (`/admin`, три раздела)
- **Заказы** — поиск (имя/телефон/№), фильтры по статусу/оплате/дате, сортировка; клик по заявке открывает карточку с составом, быстрыми кнопками связи и заметкой оператора;
- **Меню** — редактор блюд (название, цена, категория, описание, бейджи, доступность) + **загрузка фото** прямо из админки; добавление/удаление — **без правки кода**, сразу применяется на сайте;
- **обновления в реальном времени** (Server-Sent Events): новые заказы и смена статусов появляются мгновенно, без перезагрузки; в окне клиента статус тоже меняется сразу;
- **редактор заказа** прямо в карточке: данные клиента, адрес и состав (с пересчётом суммы по меню);
- клиентский статус доставки виден бейджем в списке заказов;
- в карточке заказа — **статус доставки** (принят → готовим → курьер Яндекс → доставлен), который видит клиент в трекинге, и ссылка трекинга для клиента;
- **Аналитика** — выбор периода (сегодня / 7 / 30 дней / произвольные даты) и сравнение с предыдущим периодом; честные метрики: оформлено, **выручка без отменённых**, средний чек, оплачено онлайн, выполнено, отменено; график по дням/часам, топ блюд; **экспорт заявок в CSV**. Это управленческая аналитика — не замена онлайн-кассы/фискальных чеков (54-ФЗ).

## Настройка под клиента

| Что заменить                | Где                                                          |
|-----------------------------|--------------------------------------------------------------|
| Телефон, адрес, мессенджеры | `index.html` (сейчас стоят демо-данные)                      |
| Блюда, цены                 | **в админке → вкладка «Меню»** (хранится в `data/menu.json`); сайт берёт меню из `/api/menu` |
| Фото блюд                   | папка `img/` (поле фото можно добавить в меню по аналогии)   |
| Логин/пароль админки        | `ADMIN_USER` + `ADMIN_PASS_HASH` (хеш, см. ниже) или `ADMIN_PASS` |
| Секрет сессий               | `SESSION_SECRET` (длинная случайная строка, обязательно в проде) |
| Драйвер хранилища           | `STORE_DRIVER=json` принудительно отключает SQLite           |
| Порт                        | переменная окружения `PORT` (по умолчанию 3000)              |
| Бесплатная доставка от суммы | переменная `FREE_DELIVERY_FROM` (по умолчанию 500 ₽)        |
| Домен в Open Graph          | замените `https://ВАШ-ДОМЕН` в `<head>` `index.html`         |
| Онлайн-оплата               | переменные `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`         |
| Яндекс.Метрика              | раскомментировать счётчик в `<head>`, вписать ID             |

Все переменные окружения собраны в `.env.example` (можно скопировать в `.env`).

Пример запуска со своими параметрами:

```bash
ADMIN_USER=boss ADMIN_PASS=secret123 PORT=8080 npm start
```

(на Windows в PowerShell: `$env:ADMIN_PASS="secret123"; npm start`)

## Онлайн-оплата ЮKassa

1. Зарегистрируйтесь в [ЮKassa](https://yookassa.ru) и получите `shopId` и секретный ключ.
2. Запустите сервер с ключами:

   ```bash
   # macOS/Linux
   YOOKASSA_SHOP_ID=000000 YOOKASSA_SECRET_KEY=ваш_ключ npm start
   ```
   ```powershell
   # Windows PowerShell
   $env:YOOKASSA_SHOP_ID="000000"; $env:YOOKASSA_SECRET_KEY="ваш_ключ"; npm start
   ```
3. На сайте в форме заказа появится пункт «Онлайн-картой». При оформлении клиент
   уходит на оплату ЮKassa и возвращается на `/payment-return`, где видит статус.
4. Для продакшена укажите `PUBLIC_URL` (адрес сайта) и настройте в ЮKassa
   вебхук на `POST /api/yookassa-webhook` — статусы оплат будут обновляться сами.

Без ключей онлайн-оплата просто скрыта, сайт работает в режиме «оплата при получении».
Платёжные ключи нигде в коде не хранятся — только в переменных окружения.

## Доставка через Яндекс

Доставка — курьерами Яндекс.Доставки. Два режима:

- **Ручной (работает сразу):** оператор создаёт доставку в кабинете Яндекс.Доставки и
  вставляет ссылку отслеживания в карточку заказа (блок «Яндекс.Доставка»). Клиент видит
  её в окне трекинга по кнопке «Отследить курьера в Яндексе».
- **Авто через API:** при заданных ключах в карточке заказа появляются кнопки
  «Создать заявку Яндекс» и «Обновить статус» — заявка создаётся и статусы подтягиваются автоматически,
  этап доставки клиента обновляется сам.

Переменные (см. `.env.example`): `YANDEX_DELIVERY_TOKEN`, `YANDEX_GEOCODER_KEY`
(для координат адреса клиента), `YANDEX_PICKUP_LON/LAT/ADDRESS/CONTACT/PHONE`.

> ⚠️ Автосоздание заявки (Cargo API) зависит от вашего аккаунта: формат заявки
> (класс тарифа, габариты, обязательные поля) может потребовать доработки в
> `lib/yandex-delivery.js`. Гарантированно работающий путь без настройки — ручная ссылка отслеживания.

## Безопасность

- Смените пароль админки: `npm run hash "новый-пароль"` → строку `ADMIN_PASS_HASH=...`
  положите в `.env`. Пароль в открытом виде нигде не хранится.
- Задайте `SESSION_SECRET` (длинная случайная строка).
- В продакшене включите HTTPS — тогда cookie сессии автоматически станут `secure`
  (определяется по `PUBLIC_URL=https://...`).
- Вебхук ЮKassa принимается только с IP ЮKassa (отключить для теста: `WEBHOOK_VERIFY=off`).
- Защита входа от перебора: после 5 неудачных попыток — прогрессивная блокировка по IP; при входе session-id регенерируется (защита от fixation).
- Пока используется пароль по умолчанию, в админке висит красный баннер-предупреждение.

## Деплой

Запуск в Docker, HTTPS через nginx + Let's Encrypt и чек-лист публикации — в **`DEPLOY.md`**.
Кратко:

```bash
cp .env.example .env      # задайте секреты
docker compose up -d --build
```

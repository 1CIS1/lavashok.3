# ☁️ Как разместить ЛавашОК в облаке (без своего ПК)

Цель: сайт работает 24/7 по своему адресу с HTTPS, компьютер выключать можно.
Площадка сама делает `npm install` и запускает `npm start`.

Подойдут российские облака приложений: **Amvera Cloud**, **Timeweb Cloud (Приложения)**,
**Reg.ru Cloud**, **Beget**. У всех схема похожа — ниже общий план.

---

## Что понадобится

- Аккаунт на выбранной площадке.
- Папка проекта `lavashok` (заливаем её целиком, **без** `node_modules` и `data`).
- 5 минут.

---

## Шаги

### 1. Залить код
Два способа (зависит от площадки):
- **Через GitHub:** создай репозиторий, загрузи туда папку проекта, подключи репозиторий к площадке.
- **Прямой загрузкой:** у Amvera/Timeweb можно загрузить проект через их интерфейс или их git.

> `node_modules` и `data` загружать не нужно — они в `.gitignore`. Площадка соберёт зависимости сама.

### 2. Создать приложение
- Тип: **Node.js** (версия 18 или новее).
- Команда запуска: **`npm start`**.
- Порт: приложение само берёт порт из переменной `PORT` — обычно площадка подставляет его автоматически, ничего делать не нужно.

### 3. Прописать переменные окружения
В настройках приложения (раздел Environment / Переменные) добавь:

| Переменная           | Значение                                                        |
|----------------------|-----------------------------------------------------------------|
| `SESSION_SECRET`     | любая длинная случайная строка (32+ символов)                   |
| `ADMIN_PASS_HASH`    | хеш пароля админки (как получить — ниже)                        |
| `PUBLIC_URL`         | адрес сайта, который выдаст площадка, напр. `https://lavashok.amvera.io` |
| `YOOKASSA_SHOP_ID`   | (если нужна онлайн-оплата) из кабинета ЮKassa                   |
| `YOOKASSA_SECRET_KEY`| (если нужна онлайн-оплата)                                      |
| `YANDEX_DELIVERY_TOKEN` | (если нужна авто-доставка Яндекс)                            |

`PUBLIC_URL` важен: по нему включаются защищённые cookie и формируется ссылка возврата после оплаты.

**Как получить `ADMIN_PASS_HASH`:** один раз на своём ПК в папке проекта выполни
```
npm run hash "придумай-пароль"
```
Скопируй строку `ADMIN_PASS_HASH=...` и вставь её значение в переменную на площадке.
Так пароль нигде не хранится в открытом виде.

### 4. Подключить постоянный диск (ОБЯЗАТЕЛЬНО)
Закажи у площадки **постоянный диск (Volume / Persistent storage)** и примонтируй его на путь
папки данных приложения — `./data` (или абсолютный путь к ней).

Там лежат заказы, меню, настройки, промокоды, загруженные фото. **Без постоянного диска при
каждом обновлении эти данные сотрутся.**

### 5. Запустить
Нажми Deploy / Развернуть. Через минуту-две получишь адрес вида `https://lavashok.xxx.ru`.

## После запуска

- Сайт: `https://твой-адрес/`
- Админка: `https://твой-адрес/admin` (логин `admin`, пароль — твой из `ADMIN_PASS_HASH`)
- Кухня: `https://твой-адрес/kitchen`
- В админке → **Настройки**: впиши телефон, адрес, часы. → **Акции**: промокоды/баннер. → **Меню**: блюда и фото.
- Замени `https://ВАШ-ДОМЕН` в Open Graph-тегах `index.html` на свой адрес (для красивых превью ссылок).
- Если подключил ЮKassa — в её кабинете укажи вебхук `https://твой-адрес/api/yookassa-webhook`.

---

## Частые вопросы

- **SQLite не собрался на сервере?** Не страшно — приложение само переходит на JSON-файл
  (`data/orders.json`). Главное — подключённый постоянный диск на `data`.
- **Сменить пароль позже?** Сгенерируй новый хеш (`npm run hash "..."`) и обнови `ADMIN_PASS_HASH`.
- **Своя нагрузка/трафик?** Для маленькой шаурмячной хватит самого дешёвого тарифа.

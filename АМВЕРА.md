# ☁️ Загрузка ЛавашОК на Amvera Cloud — по шагам

Проект уже подготовлен под Amvera: в корне лежит `amvera.yml` (Node 20, запуск `npm start`,
постоянный диск `/data`, порт 3000) и поддержка переменной `DATA_DIR`.

Итог: сайт будет работать 24/7 по адресу с HTTPS, компьютер можно выключать.

---

## Шаг 0. Подготовь два секрета (на своём ПК, один раз)

В папке `lavashok` в PowerShell:

```
npm run hash "придумай-пароль-админки"
```
Скопируй строку вида `ADMIN_PASS_HASH=$2a$10$....` — её значение понадобится ниже.

Сгенерируй секрет сессий:
```
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```
Скопируй полученную длинную строку — это `SESSION_SECRET`.

---

## Шаг 1. Регистрация

Зарегистрируйся на **amvera.ru** (вход по телефону/почте).

---

## Шаг 2. Создать проект

1. Нажми **«Создать проект»** (Создать → Приложение).
2. Имя проекта, например `lavashok`.
3. Тариф — подойдёт самый дешёвый/стартовый.

---

## Шаг 3. Залить код

Способ A — **через веб-интерфейс (проще)**:
- На вкладке загрузки кода перетащи **все файлы проекта**, **КРОМЕ папок `node_modules` и `data`**
  (зависимости Amvera поставит сама, данные создадутся на постоянном диске).

Способ B — **через Git** (если умеешь):
```
git init
git add .
git commit -m "lavashok"
git remote add amvera https://git.amvera.ru/ТВОЙ_ЛОГИН/lavashok
git push amvera master
```

> Файл `amvera.yml` уже в проекте — Amvera прочитает из него настройки (Node, запуск, диск, порт).
> Если Amvera предложит мастер настройки — укажи то же самое: окружение **Node.js 20**,
> команда запуска **`npm start`**, постоянный диск на **`/data`**, порт **3000**.

---

## Шаг 4. Переменные окружения

В проекте открой **«Переменные окружения»** и добавь:

| Переменная        | Значение                                                |
|-------------------|---------------------------------------------------------|
| `DATA_DIR`        | `/data`  ← данные на постоянном диске                   |
| `SESSION_SECRET`  | длинная строка из шага 0                                |
| `ADMIN_PASS_HASH` | хеш пароля из шага 0 (значение после `=`)               |
| `PUBLIC_URL`      | адрес, который выдаст Amvera, напр. `https://lavashok-логин.amvera.io` |
| `YOOKASSA_SHOP_ID` / `YOOKASSA_SECRET_KEY` | по желанию — для онлайн-оплаты          |
| `YANDEX_DELIVERY_TOKEN` | по желанию — для авто-доставки Яндекс              |

`PUBLIC_URL` точный адрес можно вписать после первого запуска (Amvera покажет домен) и пере-развернуть.

---

## Шаг 5. Постоянный диск

Убедись, что у проекта включено **постоянное хранилище (Persistent storage)** и оно
смонтировано на **`/data`** (в `amvera.yml` это уже прописано — `persistenceMount: /data`).
Вместе с `DATA_DIR=/data` это значит: заказы, меню, настройки, промокоды и фото
**сохраняются между обновлениями**.

---

## Шаг 6. Запуск

Нажми **«Собрать/Развернуть»**. Подожди 1–3 минуты (Amvera выполнит `npm install` и `npm start`).
В логах должно появиться «🌯 ЛавашОК запущен!».

Открой выданный адрес — сайт работает.

---

## После запуска

- Сайт: `https://твой-адрес/`
- Админка: `https://твой-адрес/admin` (логин `admin`, пароль — твой)
- Кухня: `https://твой-адрес/kitchen`
- В админке: **Настройки** (телефон, адрес, часы, бесплатная доставка), **Акции** (промокоды, баннер), **Меню** (блюда, фото).
- Замени `https://ВАШ-ДОМЕН` в Open Graph-тегах `index.html` на свой адрес (для красивых превью ссылок в мессенджерах) и пере-разверни.
- Если включил ЮKassa — в её кабинете укажи вебхук `https://твой-адрес/api/yookassa-webhook`.

## Если что-то не так

- **Приложение не стартует / порт.** Посмотри логи Amvera. Наше приложение слушает порт из
  `PORT` (по умолчанию 3000) и в `amvera.yml` указан `containerPort: 3000`. Если Amvera ждёт
  другой порт — приведи `containerPort` и переменную `PORT` к одному значению.
- **Данные пропали после обновления.** Значит постоянный диск не подключён или `DATA_DIR`
  не равен пути монтирования. Проверь Шаг 4 и Шаг 5.
- **`better-sqlite3` не собрался.** Это нормально — приложение само работает на JSON-файле
  (`/data/orders.json`). На функциональность не влияет.
- **Не заходит в админку (разлогинивает).** Проверь, что задан `SESSION_SECRET` и `PUBLIC_URL`
  начинается с `https://`.

# 🔧 Починить загрузку на Amvera (package.json в корне)

**Проблема:** файлы попали внутрь папки `lavashok1`, а `package.json` должен лежать
в **корне** репозитория. Из-за этого npm пишет `ENOENT: ... /source/package.json` и сборка
падает (exit 254).

**Цель:** чтобы в корне репозитория Amvera лежали `package.json`, `server.js`, `index.html`,
`amvera.yml`, папки `lib`, `assets` и т.д. — без обёртки `lavashok1`.

Ниже — надёжный способ через Git.

---

## Шаг 0. Проверить, установлен ли Git

1. Открой **PowerShell** (Пуск → набери «PowerShell» → Enter).
2. Введи и нажми Enter:
   ```
   git --version
   ```
3. Если показало версию (например `git version 2.43.0`) — Git есть, иди к Шагу 1.
4. Если «git не является командой» — установи Git:
   - Скачай с **https://git-scm.com/download/win**
   - Запусти установщик, жми **Next** до конца (ничего менять не нужно).
   - **Закрой и снова открой PowerShell.**

---

## Шаг 1. Перейти в папку проекта

```
cd C:\Users\mzolo\Desktop\claude\lavashok
```
Проверь, что ты в правильной папке:
```
dir
```
В списке должны быть **package.json**, **server.js**, **index.html**. Если их нет — ты не в той папке.

---

## Шаг 2. Подготовить файлы (вводи по одной команде, Enter после каждой)

```
git init
```
Создаёт локальный репозиторий (может написать «Reinitialized» — это нормально).

```
git add .
```
Добавляет все файлы. `node_modules` и `data` не добавятся — они в `.gitignore`, и это правильно.

```
git commit -m "lavashok"
```
Сохраняет снимок файлов.
> Если попросит указать имя и почту (только в первый раз), введи две команды и повтори commit:
> ```
> git config --global user.email "you@example.com"
> git config --global user.name "Ваше Имя"
> git commit -m "lavashok"
> ```

```
git branch -M master
```
Называет ветку `master` (так ждёт Amvera).

---

## Шаг 3. Привязать репозиторий Amvera

```
git remote add amvera https://git.amvera.ru/maksimlavash/lavashok1
```
> Если ошибка «remote amvera already exists» — введи вместо неё:
> ```
> git remote set-url amvera https://git.amvera.ru/maksimlavash/lavashok1
> ```

---

## Шаг 4. Отправить файлы в Amvera

```
git push amvera master --force
```
- Появится запрос **логина и пароля от аккаунта Amvera** — введи их.
  Пароль при вводе **не отображается** (символы не видны) — это нормально, печатай и жми Enter.
- Дождись завершения — увидишь строки про объекты и `master -> master`.

`--force` нужен, потому что в репозитории уже есть коммит конфигурации.

---

## Шаг 5. Проверить и пересобрать

1. В кабинете Amvera открой проект → **дерево файлов**.
   В **корне** теперь должен быть `package.json` (а не папка `lavashok1`).
2. Нажми **«Пересобрать» / «Развернуть»**.
3. В логах сборки должно появиться **«🌯 ЛавашОК запущен!»**.

---

## Если не получается

- **`git push` отклонён (авторизация).** Проверь логин/пароль Amvera. Возможно, нужен
  специальный пароль/токен — посмотри в настройках аккаунта Amvera раздел про Git-доступ.
- **Ругается на ветку.** Попробуй: `git push amvera HEAD:master --force`
- **Снова ENOENT package.json.** Пришли первые 20 строк нового лога сборки — посмотрю.

---

## Способ без Git (перезалить через веб)

1. В Amvera удали папку `lavashok1` (и `.git`, если видно).
2. На ПК **открой** папку `lavashok`, внутри неё нажми **Ctrl+A** (выделить всё содержимое).
3. Перетащи в окно загрузки Amvera **выделенные файлы и папки**, а **не саму папку** `lavashok`.
   В загрузку должны попасть сразу `package.json`, `server.js`, `lib`, `assets`… без обёртки.
4. Пересобери.

> Почему так: если перетащить папку целиком, Amvera создаёт её как подпапку (`lavashok1`),
> и `package.json` оказывается на уровень глубже, чем нужно.

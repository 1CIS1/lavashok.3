# 📕 ЛавашОК — полная инструкция: установка, запуск, работа

Этот файл — всё в одном месте. Читай сверху вниз, по шагам. Рассчитано на Windows.

---

## 0. Что это вообще такое

Это готовый сайт доставки шаурмы **+ программа-сервер**, которая:
- показывает сайт посетителям;
- принимает заявки с формы «Оформить заказ»;
- складывает их в базу;
- показывает все заявки в админке, где можно менять статусы.

Сайт и админка работают на твоём компьютере по адресу `http://localhost:3000`.
«localhost» = «этот же компьютер». Никто из интернета пока его не видит — это нормально
для разработки и демонстрации.

## 1. Что нужно установить один раз (Node.js)

Программа написана на Node.js. Это бесплатно. Ставится один раз.

1. Открой сайт **https://nodejs.org**
2. Скачай большую зелёную кнопку **LTS** (рекомендуемая версия).
3. Запусти скачанный файл `.msi` и жми «Далее» до конца (ничего менять не надо).
4. Проверь, что установилось. Нажми `Win + R`, впиши `cmd`, Enter. В чёрном окне набери:
   ```
   node --version
   ```
   Если появилось что-то вроде `v20.11.0` — всё хорошо. Если «не является командой» —
   перезагрузи компьютер и проверь снова.

> Достаточно версии 18 или новее. LTS всегда подходит.

## 3. Первый запуск (делается один раз при установке)

### Шаг 3.1 — открыть командную строку в папке проекта

1. Открой папку `lavashok` в Проводнике.
2. В адресной строке Проводника (сверху, где путь к папке) сотри текст, впиши `powershell`
   и нажми Enter. Откроется синее окно PowerShell — уже **внутри нужной папки**.

   *Альтернатива:* зажми `Shift`, кликни правой кнопкой по пустому месту в папке →
   «Открыть окно PowerShell здесь».

### Шаг 3.2 — установить зависимости

В открывшемся окне набери и нажми Enter:
```
npm install
```
Подожди 10–30 секунд. Появится папка `node_modules` — так и должно быть.
Это делается **только один раз** (и ещё раз, если перенёс проект на другой компьютер).

> #### ⚠️ Если вылезла ошибка «Невозможно загрузить файл npm.ps1… выполнение сценариев отключено»
>
> Это значит, что Node.js установлен, но Windows из соображений безопасности
> запрещает PowerShell запускать скрипты. Лечится один раз. Выполни в том же окне:
> ```
> Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
> ```
> На вопрос подтверждения нажми клавишу `Y` и Enter. После этого снова `npm install`.
> Эта настройка безопасна (разрешает только локальные скрипты) и рекомендуется самой Microsoft.
>
> **Не хочешь менять настройку?** Просто добавляй `.cmd` к командам — блокировки не будет:
> `npm.cmd install`, потом `npm.cmd start`.

### Шаг 3.3 — запустить
```
npm start
```
Если всё хорошо, увидишь в окне:
```
  🌯  ЛавашОК запущен!
  ─────────────────────────────────────────────
  Сайт:    http://localhost:3000
  Админка: http://localhost:3000/admin
  Логин:   admin  /  пароль: lavashok
  ─────────────────────────────────────────────
```

**Готово. Сервер работает.** Окно PowerShell не закрывай — пока оно открыто, сайт работает.

## 4. Как открыть сайт и админку

Открой браузер (Chrome, Edge — любой) и введи в адресную строку:

| Что     | Адрес                          |
|---------|--------------------------------|
| Сайт    | `http://localhost:3000`        |
| Админка | `http://localhost:3000/admin`  |

При входе в админку откроется страница входа — введи:
- логин: **admin**
- пароль: **lavashok**

Чтобы выйти — кнопка «Выйти» в правом верхнем углу админки.

> Заявки теперь хранятся в базе **SQLite** (`data\orders.db`). Если на твоём
> компьютере SQLite не установится — сайт сам переключится на файл `orders.json`,
> ничего делать не нужно.

## 5. Как это работает каждый день

1. **Утром:** открой папку проекта → запусти PowerShell (шаг 3.1) → набери `npm start`.
   (`npm install` повторять не нужно — только в самый первый раз.)
2. Открой админку `http://localhost:3000/admin` в браузере — оставь вкладку открытой.
3. Когда клиент оставляет заявку на сайте — она **сама появляется в админке**
   (список обновляется каждые 20 секунд, или жми кнопку «Обновить»).
4. В колонке «Статус» меняй состояние заявки:
   - **Новый** — только что пришла;
   - **Перезвонили** — связались с клиентом;
   - **Готово** — заказ выполнен;
   - **Отменён** — клиент отказался.
5. В заявке видно **состав корзины, сумму и способ оплаты** (при получении или онлайн).
   Телефон кликабельный — нажатие открывает звонок (если настроена телефония).

### Как остановить сервер
В окне PowerShell нажми `Ctrl + C`. Сервер остановится, сайт перестанет открываться.
Чтобы снова запустить — опять `npm start`.

### Где лежат заявки
Все заявки хранятся в файле `data\orders.json` внутри папки проекта.
Даже если закрыть сервер — заявки не пропадут, при следующем запуске будут на месте.

## 6. Как менять содержимое сайта

Все тексты сайта — в файле `index.html`. Открывай его Блокнотом или, лучше,
бесплатным редактором **VS Code** (https://code.visualstudio.com). Найди нужное
(`Ctrl + F`) и замени. После изменений обнови страницу сайта в браузере (`F5`).

| Что поменять             | Что искать в `index.html`                         |
|--------------------------|---------------------------------------------------|
| Телефон                  | `+7 (495) 120-45-67` и `tel:+74951204567`         |
| Адрес                    | `г. Москва, ул. Арбат, д. 24`                      |
| WhatsApp / Telegram      | `wa.me/74951204567` и `t.me/lavashok`             |
| Время доставки, зоны     | блок `ДОСТАВКА`                                    |
| Отзывы                   | блок `ОТЗЫВЫ`                                      |

> 🍽 **Блюда и цены меняются прямо в админке** — раздел «Меню» (см. ниже).
> Править код больше не нужно: меню хранится в `data\menu.json`, а сайт
> подгружает его сам. Сервер считает суммы заказов по этим же ценам.

### Управление меню (в админке)
Зайди в админку → вкладка **«Меню»**. Там можно:
- менять название, цену, категорию, описание и доступность блюда;
- добавлять новые блюда (кнопка «+ Добавить блюдо») и удалять (значок 🗑);
- снять галочку «В продаже» — блюдо скроется с сайта, но не удалится.

После правок жми **«Сохранить меню»** — изменения сразу появятся на сайте
(посетителю достаточно обновить страницу).

### Что ещё умеет админка
- **Заказы**: поиск по имени/телефону/номеру, фильтры по статусу, оплате и дате,
  сортировка. Клик по заявке открывает карточку с составом, кнопками «Позвонить»
  и WhatsApp и полем для заметки.
- **Аналитика**: сумма заказов, средний чек, выручка по дням, топ блюд.
- **Экспорт**: кнопка «Экспорт CSV» выгружает заявки — файл открывается в Excel.

### Сменить пароль от админки
Пароль по умолчанию `admin` / `lavashok`. Чтобы задать свой, запускай сервер так
(в PowerShell, в папке проекта):
```
$env:ADMIN_USER="boss"; $env:ADMIN_PASS="мой-пароль"; npm start
```
Тогда вход в админку будет под `boss` / `мой-пароль`. Без этой команды работают
значения по умолчанию.

### Добавить фото блюд
Положи картинки в папку `img` внутри проекта. В `index.html` рядом с каждым блюдом
есть подсказка-комментарий, как заменить эмодзи на фото (`<img src="img/...">`).

### Включить онлайн-оплату (ЮKassa)
По умолчанию работает оплата при получении. Чтобы принимать оплату картой онлайн:

1. Зарегистрируйся в **ЮKassa** (yookassa.ru) и получи `shopId` и секретный ключ.
2. Запусти сервер с ключами (PowerShell, в папке проекта):
   ```
   $env:YOOKASSA_SHOP_ID="номер_магазина"; $env:YOOKASSA_SECRET_KEY="секретный_ключ"; npm start
   ```
3. На сайте в форме заказа появится пункт «💳 Онлайн-картой». Клиент платит на
   странице ЮKassa и возвращается на сайт, где видит статус оплаты.

> Ключи нигде не сохраняются в файлах — они живут только в этой команде запуска.
> Я (ассистент) ключи не запрашиваю и не ввожу: их вводишь только ты.

## 7. Как показать / продать (демо за 1 минуту)

1. Запусти сервер (`npm start`).
2. Открой сайт `http://localhost:3000`, покажи: меню, фильтры по категориям,
   адаптивность (сожми окно браузера — вёрстка перестроится под телефон).
3. Пролистай вниз до «Оформить заказ», заполни имя и телефон, отправь.
   Покажи экран «Заявка принята».
4. Открой соседнюю вкладку с админкой `http://localhost:3000/admin` —
   заявка уже там. Поменяй статус «Новый» → «Перезвонили».
5. Вывод: «Сайт принимает заказы, владелец видит их в реальном времени и ведёт по статусам».

## 8. Если что-то не работает

| Проблема                                   | Что делать                                                        |
|--------------------------------------------|-------------------------------------------------------------------|
| `npm` не является командой                 | Node.js не установлен или нет перезагрузки. Вернись к шагу 1.      |
| `Невозможно загрузить файл npm.ps1, выполнение сценариев отключено` | PowerShell блокирует скрипты. Один раз выполни команду ниже и подтверди клавишей `Y`. См. раздел «Ошибка с npm.ps1». |
| `Cannot find module 'express'`             | Не сделан `npm install`. Выполни его в папке проекта (шаг 3.2).    |
| `EADDRINUSE` / порт занят                  | Уже запущен другой сервер. Закрой его (`Ctrl + C`) или запусти так: `$env:PORT="3001"; npm start`, и открывай `http://localhost:3001`. |
| Сайт не открывается                        | Проверь, что окно PowerShell с `npm start` открыто и не закрыто.   |
| Заявки не появляются в админке             | Нажми «Обновить» в админке. Проверь, что сервер запущен.           |
| Браузер не спрашивает пароль в админке     | Открой админку в новом окне браузера / режиме инкогнито.           |
| Окно PowerShell закрыл — сайт пропал       | Это нормально. Запусти заново: PowerShell в папке → `npm start`.   |

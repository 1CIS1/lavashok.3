# ☁️ Размещение ЛавашОК на Timeweb Cloud (Приложения) — по шагам

Простой путь: код кладём на GitHub, Timeweb сам собирает и запускает, выдаёт адрес с HTTPS.
Командная строка не нужна — всё через сайт.

> ⚠️ Важно про данные. На «Облачных приложениях» файловая система может сбрасываться при
> пересборке — заказы, меню и настройки (они в файлах) могут не сохраниться между обновлениями
> кода. Для **показа клиента и теста** это нормально. Для **постоянной работы** лучше VPS +
> Docker (данные на диске) — см. `DEPLOY.md`.

---

## Часть 1. Положить код на GitHub (через сайт, без программ)

1. Зарегистрируйся на **github.com** (если ещё нет аккаунта).
2. Нажми **New repository** (создать репозиторий):
   - Имя: `lavashok`
   - Тип: **Private** (приватный) или Public — на твой выбор.
   - Нажми **Create repository**.
3. На странице нового репозитория нажми ссылку **«uploading an existing file»**
   (или **Add file → Upload files**).
4. На своём ПК **открой** папку `lavashok` и зайди **внутрь неё**.
5. Выдели **всё содержимое** папки (Ctrl+A): файлы `package.json`, `server.js`, `index.html`,
   папки `lib`, `assets`, `nginx` и т.д.
   **Не выделяй** папки `node_modules` и `data` (их грузить не нужно).
6. Перетащи выделенное в окно загрузки GitHub.
   > Ключевой момент: тащи **содержимое** папки, а не саму папку `lavashok`.
   > В корне репозитория должен оказаться `package.json` — без обёртки.
7. Внизу нажми **Commit changes**.
8. Проверь: на главной странице репозитория в списке файлов виден **`package.json`** (а не одна папка).

---

## Часть 2. Создать приложение в Timeweb Cloud

1. Войди в **timeweb.cloud** → раздел **«Приложения»** (Apps) → **«Создать приложение»**.
2. Тип приложения: **Backend** → фреймворк **Node.js**.
3. Источник кода: **GitHub** → разреши доступ и выбери репозиторий **`lavashok`**, ветку `main` (или `master`).

---

## Часть 3. Настройки сборки и запуска

В настройках приложения укажи (если мастер спросит):
- **Команда сборки (build):** `npm install`
- **Команда запуска (start):** `npm start`
- **Версия Node.js:** 18 или новее (20 — хорошо).
- **Порт:** приложение само берёт порт из переменной `PORT`, которую Timeweb подставит. Обычно ничего вручную указывать не нужно.

---

## Часть 4. Переменные окружения

В разделе **«Переменные окружения»** добавь:

| Переменная        | Значение                                                       |
|-------------------|----------------------------------------------------------------|
| `SESSION_SECRET`  | длинная случайная строка (32+ символов)                        |
| `ADMIN_PASS_HASH` | хеш пароля админки (как получить — ниже)                       |
| `PUBLIC_URL`      | адрес, который выдаст Timeweb, напр. `https://lavashok-xxx.twc1.net` |
| `YOOKASSA_SHOP_ID` / `YOOKASSA_SECRET_KEY` | по желанию — онлайн-оплата                    |
| `YANDEX_DELIVERY_TOKEN` | по желанию — авто-доставка Яндекс                        |

**Как получить `ADMIN_PASS_HASH`** (на своём ПК, один раз, в папке проекта):
```
npm run hash "придумай-пароль"
```
Скопируй значение после `ADMIN_PASS_HASH=` и вставь в переменную.

`PUBLIC_URL` точный адрес можно вписать после первого запуска (Timeweb покажет домен) и пере-развернуть.

---

## Часть 5. Запуск

Нажми **«Развернуть» / Deploy**. Через 1–3 минуты Timeweb выполнит `npm install` и `npm start`.
В логах появится **«🌯 ЛавашОК запущен!»**, и ты получишь адрес.

---

## После запуска

- Сайт: `https://твой-адрес/`
- Админка: `https://твой-адрес/admin` (логин `admin`, пароль — твой)
- Кухня: `https://твой-адрес/kitchen`
- В админке: **Настройки** (телефон, адрес, часы), **Акции** (промокоды/баннер), **Меню** (блюда/фото).
- Если включил ЮKassa — в её кабинете укажи вебхук `https://твой-адрес/api/yookassa-webhook`.

---

## Если что-то не так

- **Ошибка `ENOENT package.json`** → в репозитории `package.json` оказался внутри подпапки.
  Загрузи файлы заново так, чтобы `package.json` был в **корне** репозитория (Часть 1, шаги 4–6).
- **Не открывается / порт** → приложение слушает порт из `PORT`. Убедись, что в настройках Timeweb
  команда запуска `npm start` и ничего не переопределяет порт вручную.
- **Разлогинивает из админки** → проверь, что задан `SESSION_SECRET` и `PUBLIC_URL` начинается с `https://`.
- **Данные пропали после пересборки** → это особенность Apps. Для постоянного хранения заказов
  переезжай на VPS + Docker (`DEPLOY.md`) — там данные на диске.
